Database Schema

For this assignment, I partnered with Fumiko Aoki to design the database schema.

We decided to use six tables to store all of the data in the Grad Progress assignment.

Below is our table for storing Advisors:

aid sid
Primary Key Primary Key / Foreign Key

MySQL Command: CREATE TABLE `Grad_Prog_V3`.`Advisors` (`aid` INT NOT NULL,`sid` INT NOT NULL,PRIMARY KEY (`aid`, `sid`)) COMMENT = 'Maps advisors to students';

Below is our table for storing Users:

uid name position
Primary Key

MySQL Command:CREATE TABLE `Grad_Prog_V3`.`Users` (`uid` INT NOT NULL AUTO_INCREMENT, `name` CHAR(30) NOT NULL, `position` CHAR(1) NOT NULL, PRIMARY KEY (`uid`), UNIQUE INDEX `uid_UNIQUE` (`uid` ASC)) COMMENT = 'Represents all users in the system';

Below is our table for storing Student Information:

uid degree track semester_admitted
Primary Key

MySQL Command:CREATE TABLE `Grad_Prog_V3`.`Students` (`uid` INT NOT NULL, `degree` CHAR(15) NOT NULL, `track` CHAR(30) NOT NULL, `semester_admitted` VARCHAR(20) NOT NULL, PRIMARY KEY (`uid`), UNIQUE INDEX `uid_UNIQUE` (`uid` ASC)) COMMENT = 'Holds information about the student.';

Below is our table for storing Student Forms:

fid uid date meets_requirements progress_description student_signed student_signed_date advisor_signed advisor_signed_date modified_date
Primary Key Primary Key / Foreign Key

MySQL Command: CREATE TABLE `Grad_Prog_V3`.`Forms` (`fid` INT NOT NULL, `uid` INT NOT NULL, `date` DATE NOT NULL, `meets_requirements` TINYINT(1) NOT NULL, `progress_description` CHAR(200) NOT NULL, `student_signed` TINYINT(1) NOT NULL, `student_signed_date` DATE NOT NULL, `advisor_signed` TINYINT(1) NOT NULL, `advisor_signed_date` DATE NOT NULL, `modified_date` DATE NOT NULL, PRIMARY KEY (`fid`, `uid`)) COMMENT = 'Contains student progress forms';

Below is our table for mapping students to committee members

sid facultyid
Primary Key Primary Key

MySQL Command: CREATE TABLE `Grad_Prog_V3`.`Committee` (`sid` INT NOT NULL, `facultyid` INT NOT NULL, PRIMARY KEY (`sid`, `facultyid`)) COMMENT = 'Maping from student to committee member';

Below is our table for storing progress activities

sid activity date_completed
Primary Key Primary Key

MySQL Command: CREATE TABLE `Grad_Prog_V3`.`Activities` (`sid` INT NOT NULL, `activity` CHAR(100) NOT NULL, `date_completed` VARCHAR(20) NOT NULL, PRIMARY KEY (`sid`, `activity`)) COMMENT = 'Represents different completed activites in the progress form';
